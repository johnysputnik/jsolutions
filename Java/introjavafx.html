Introduction

This is set of notes for a quick start for JavaFX, using an MVVM pattern to create the simplest of apps.

The View Model

The view model holds the code for UI behaviour, so that all the view needs to be is presentation layout and bindings. The view model is as follows: 

 1: package sample;
 2: 
 3: import javafx.beans.property.ReadOnlyStringProperty;
 4: import javafx.beans.property.SimpleStringProperty;
 5: import javafx.beans.property.StringProperty;
 6: 
 7: public class SampleViewModel {
 8: 
 9:         private final StringProperty textValue = new SimpleStringProperty();
10: 
11:         public StringProperty textValueProperty() {
12:                 return textValue;
13:         }
14: 
15:         public ReadOnlyStringProperty echoTextValueProperty(){
16:                 return textValue;
17:         }
18: 
19:         public void clearText() {
20:                 textValue.setValue("");
21:         }
22: }

The View

The view then has to be defined in 2 files.

The first file is the Controller class that defines the fields and the bindings to the View Model. Ideally the View Model would be injected in a constructor or via some other property. 

23: package sample;
24: 
25: import javafx.event.ActionEvent;
26: import javafx.fxml.FXML;
27: import javafx.scene.control.Label;
28: import javafx.scene.control.TextField;
29: 
30: 
31: public class SampleView {
32: 
33:         SampleViewModel vm = new SampleViewModel();
34: 
35:         @FXML
36:         private TextField textField;
37: 
38:         @FXML
39:         private Label echoTextField;
40: 
41:         @FXML
42:         void initialize() {
43: 
44:                 textField.textProperty().bindBidirectional(vm.textValueProperty());
45:                 echoTextField.textProperty().bind(vm.echoTextValueProperty());
46:         }
47: 
48:         public void clearText(ActionEvent actionEvent) {
49:                 vm.clearText();
50:         }
51: }

The layout of the view is then defined in a FXML file: 

52: <?xml version="1.0" encoding="UTF-8"?>
53: 
54: <?import javafx.scene.control.TextField?>
55: <?import javafx.scene.layout.*?>
56: 
57: <?import javafx.scene.control.Button?>
58: <?import javafx.scene.control.Label?>
59: <GridPane xmlns:fx="http://javafx.com/fxml/1" xmlns="http://javafx.com/javafx/8"
60:           alignment="center" hgap="10" vgap="10" fx:controller="sample.SampleView">
61:    <children>
62:       <TextField GridPane.rowIndex="0" fx:id="textField" />
63:       <Label GridPane.rowIndex="1" fx:id="echoTextField"/>
64:       <Button GridPane.rowIndex="2" fx:id="clearButton" onAction="#clearText">Clear</Button>
65:    </children>
66:    <columnConstraints>
67:       <ColumnConstraints />
68:    </columnConstraints>
69:    <rowConstraints>
70:       <RowConstraints />
71:       <RowConstraints />
72:    </rowConstraints>
73: </GridPane>

This references the code behind class and properties. Ideally it would be good if these two files could be merged a bit more and the bindings could be set in the FXML itself.

Main

The main function is then defined as follows: 

74: package sample;
75: 
76: import javafx.application.Application;
77: import javafx.fxml.FXMLLoader;
78: import javafx.scene.Parent;
79: import javafx.scene.Scene;
80: import javafx.stage.Stage;
81: 
82: public class Main extends Application {
83: 
84:         @Override
85:         public void start(Stage primaryStage) throws Exception{
86:                 Parent root = FXMLLoader.load(getClass().getResource("SampleView.fxml"));
87:                 primaryStage.setTitle("Hello World");
88:                 primaryStage.setScene(new Scene(root, 300, 275));
89:                 primaryStage.show();
90:         }
91: 
92: 
93:         public static void main(String[] args) {
94:                 launch(args);
95:         }
96: }
